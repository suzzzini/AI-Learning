# SSH
## SSH 란
SSH(SecureShell)는 원격 컴퓨터와 **암호화된 통신** 을 하기 위한 프로토콜이다.

예전에는 ID/Password를 직접 물어봤는데, 이 방식은 도청, 유출, 재사용 될 위험이 있기 때문에
공개키/개인키 방식으로 사용자 인증을 한다.

1. 사용자 A: Public Key, Private Key 생성
2. 사용자 A: Public key를 서버에 전달
3. 사용자 A: Private Key로 서명
4. 서버: Public Key로 검증

## GitHub SSH 연결 예시
예시로, 
Git, VS Code를 세팅하고 터미널에서 처음 git push 할 때

Username:
Password:

를 계속 물어봤다.

그 이유는 GitHub는 내가 해당 Repository 주인이 맞는지를 확인해야 하기 때문이다.

그래서 설정해준 것이 SSH 설정

1. 공개키, 개인키 생성
 $ ssh-keygen
 -> id_ed25519(개인키)
 -> id_ed25519.pub(공개키)

2. GitHub에 생성한 공개키 등록
 -> id_ed25519.pub
 
3. 이후 git push 를 하면 내 PC가 갖고 있는 개인키로 나를 증명하고, GitHub는 등록된 공개키로 이를 검증해서 로그인 없이 안전하게 인증을 해줌
 $ git push 


git init
- 현재 폴더를 Git 저장소로 만든다
- .git 폴더가 생김
  - 이 안에 변경 이력, 브랜치, 커밋, 설정 등이 모두 저장됨
- 프로젝트 처음 만들 때 딱 한 번 사용

git add
- 커밋할 파일은 준비(stage)한다.
- git add . // 현재 폴더의 모든 변경사항

git commit
- 변경사항을 하나의 버전으로 저장한다.
- git commit -m "설명"
  - -m : aptlwl

git push
- 내 컴퓨터의 커밋을 GitHub로 업로드
- $ git push origin main

ssh-keygen
- github에 로그인할 ssh 키 생성
- 1) ssh-keygen -t ed25519 -C "your@email.com"
- 2) ~/.ssh
- 3) 해당 폴더 안에 id_ed25519/id_ed25519.pub 키가 생성됨

git remote
- GitHub 저장소와 연결한다.
  - Github 에 https://github.com/me/AI-study 저장소가 있을 때, 
  - $ git remote add orgin git@github.com:me/AI-study.git
  - $ git remote -v // 확인
