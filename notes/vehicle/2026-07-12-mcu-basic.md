# CPU

## CPU란?
CPU(Central Processing Unit(중앙 처리 장치))는 컴퓨터의 '두뇌'이다.

프로그램이 실행될 때
- 계산을 수행하고
- 명령을 해석하고
- 다른 부품(RAM, SSD 등)을 제어하는 역할을 한다.

쉽게 말하면
    CPU는 "생각하고 일하는 사람" 이다.

## 왜 CPU가 필요할까?
예를 들어 아래와 같은 Python 코드가 있다.
``` code
speed = 100
```
컴퓨터는 speed = 100 의 의미를 전혀 모른다.
그래서 누군가가 이 *** 명령을 해석하고 실행 *** 해야 한다.
그 역할을 하는 것이 CPU 이다.

## Python 코드 실행 과정
작성한 코드
``` code
speed = 100
print(speed)
```
실행 과정
``` text
① Python 코드 작성
        │
        ▼
② Python Interpreter 가 python 코드 해석
        │
        ▼
③ OS를 통해 CPU 명령을 요청
        │
        ▼
④ CPU가 명령 실행
        │
        ▼
⑤ RAM에 변수 저장
        │
        ▼
⑥ print() 실행
        │
        ▼
⑦ 화면 출력
```
여기서 CPU는 
"RAM에 100을 저장해" 
"RAM에서 speed를 읽어와"
"출력 장치(터미널)에게 100을 출력해"
라는 명령을 실행한다. 

## CPU의 역할
CPU가 하는 일은 크게 두 가지이다.
### 1. 연산(ALU)
프로그램에서 필요한 산술 연산과 논리 연산을 수행

예를 들어, a=3+5 가 있을 때,
CPU는 3+5를 계산해서 8을 만든다.
이 계산을 담당하는 부분이 ALU(Arithmetic Logit Unit)이다.

### 2. 제어(Control Unit)
컴퓨터의 메모리, 그래픽 카드, 저장장치 등 다른 부품들이 원활하게 작동하도록 지시하고 관리

예를 들어, 
speed = 100

CPU는 RAM에 저장해. 다음 명령 읽어. print() 실행해. 처럼 전체 순서를 지시함

## CPU의 구성요소
### 제어 장치(Control Unit)
- 명령 처리를 관리, CPU 내부 및 다른 컴퓨터 구성 요소들 간 데이터 흐름을 조정

### 레지스터
- CPU 내 소형 고속 메모리 저장 위치
  -> CPU가 현재 실행 중인 명령의 데이터나 주소를 잠시 저장하는 공간
- CPU에서 현재 작업 중인 데이터를 보관하므로 데이터에 빠르게 엑세스 할 수 있음
* 레지스터 종류:
  * 범용 레지스터 : 운영 데이터 보관
  * 명령 레지스터 : 현재 처리 중인 명령 보관
  * 프로그램 카운터 : 다음에 가져올 명령어의 메모리 주소 보관
- 레지스터는 RAM 또는 캐시 메모리 등 다른 메모리 수준보다 액세스 시간이 빠름

### 연산(ALU) 
* 수행 명령 종류:
  * 데이터에 대한 기본 산술 연산(+,-,*,/)
  * 논리 연산(AND, OR, NOT)
- CPU 내 레지스터로부터 데이터를 받아 제어 장치의 명령에 따라 처리하고 결과를 생성함
  
### 메모리 관리 장치
- CPU 아키텍처에 따라 별도의 버스 인터페이스 장치 또는 메모리 관리 장치가 있을 수 있음
- 이 구성요소는 CPU, RAM의 상호 작용 관리 등 메모리 관련 작업을 처리함
- 또한 CPU 내에 있는 작고 빠른 메모리 유닛인 캐시 메모리와 데이터 처리에 CPU에 필요한 가상 메모리도 처리함

### Clock
- CPU는 클럭 신호를 사용하여 내부 작업을 동기화 함
- 클럭은 특정 주파수에서 일정한 펄스를 생성하며, 이러한 클럭 사이클은 CPU의 작업을 조정함
- 클럭 속도 츨정 단위 : 헤르츠(Hz), CPU에서 실행할 수 있는 초당 명령 수를 결정

## CPU 작동 방식
- 사이클 단위로 작동
  
    명령어 가져오기(Fetch)
    명령어 해석하기(Decode)
    명령 실행하기(Execute)
    다음 명령

## 컴퓨터에 메모리가 필요한 이유
컴퓨터는
- 프로그램
- 변수
- 사진
- 문서
등 수많은 데이터를 저장해야 함

하지만 모든 데이터를 같은 공간에 저장하지 않음
용도에 따라 다른 메모리를 사용함

대표적으로 Flash Memory, RAM, Register 가 있음
### Flash Memory
전원을 꺼도 데이터가 사라지지 않는 메모리
즉, 비휘발성 메모리

대표적으로
- 운영체제
- 프로그램
- 사진
- 문서
처럼 오랫동안 보관해야 하는 데이터가 저장되어 있음

특징:
 - 전원을 꺼도 데이터 유지
 - 저장 용량이 큼
 - RAM 보다 느림
 - 프로그램을 저장하는 공간

SSD는 Flash Memory를 이용해 만든 저장장치

### RAM(Random Access Memory)
현재 실행 중인 프로그램과 데이터를 저장하는 메모리


CPU는 프로그램을 저장하지 않고, *** 실행 *** 만 한다.
그래서 speed = 100 을 실행하면,
CPU가 RAM에 Speed 100 을 저장해. 라고 명령한다.

RAM
``` text
┌──────────────┐
│ speed =100   │
└──────────────┘
```

특징 :
 - 읽기/쓰기 빠름
 - CPU가 직접 사용
 - 전원을 끄면 모두 삭제됨
 - 실행 중인 프로그램 저장

RAM 영역
    프로그램 코드
    변수
    Stack
    Heap


### Register

# MCU(Microcontroller Unit)
## MCU란
MCU(Mmicrocontroller Unit)는 CPU, RAM, Flash, 입출력 장치(I/O)를 하나의 칩에 넣은 작은 컴퓨터이다.

쉽게 말하면, 
    특정 기능을 수행하기 위한 초소형 컴퓨터
이다.

예를 들어 자동차에는
 - 창문 올리기
 - 와이퍼 동작
 - 헤드램프 제어
 - 엔진 제어
 - 배터리 관리

같은 기능이 있다.

이러한 기능을 수행하는 ECU 내부에는 대부분 MCU가 들어 있다.

## 왜 MCU가 만들어졌을까?
우리가 사용하는 PC는

CPU, RAM, SSD, GPU, 메인보드 등이 모두 따로 있다.

하지만 자동차 문 하나를 올리는 ECU에
    Intel CPU
    16GB RAM
    1TB SSD
를 넣는다면 엄청 비싸고 전기도 많이 먹고 크기도 커진다.

그래서 필요한 기능만 수행하도록 만든 것이 MCU이다.

## MCU 내부 구조
MCU는 하나의 칩 안에
``` text
            MCU

┌────────────────────────────┐

        CPU Core

            │

 ┌──────────┼──────────┐

 │          │          │

Flash      RAM      EEPROM

 │          │

 └──────┬───┘

        │

        GPIO

        UART Controller

        SPI Controller

        I2C Controller

        CAN Controller

        Ethernet Controller

        Timer

        ADC

        PWM

└────────────────────────────┘

``` 
가 모두 들어 있다.

* Controller 란
  특정 하드웨어를 제어하는 전용 회로(IP, Peripheral)

  
## MCU 구성 요소
### 1. CPU
프로그램을 실행하는 역할을 한다.
 - 명령 해석
 - 계산
 - 메모리 접근

### 2. Flash Memory
비휘발성 메모리로 프로그램이 저장된다.

### 3. RAM
변수, 코드 등 실행 중인 데이터가 저장된다.
휘발성 메모리이다.

### 4. EEPROM
전원을 꺼도 유지되어야 하는 작은 데이터를 저장한다.
차량 VIN, 주행거리, 캘리브레이션 값 등을 저장한다.
(※ 최근 MCU는 별도의 EEPROM 대신 Flash의 일부 영역을 사용하는 경우도 많다.)

### 5. GPIO
외부 장치를 연결하는 핀이다.
예를 들어, 
    LED ON
    스위치 입력
    릴레이 제어
등을 한다.

### 6. CAN Controller 
자동차에서는 ECU끼리 통신해야 한다.
이를 위해, MCU 안에는
CAN Controller가 포함되는 경우가 많다.

``` text
Engine ECU
↓
CAN
↓
Battery ECU
↓
CAN
↓
Gateway ECU
```

CPU는
CAN_Send(message);
처럼 명령만 내리고,
실제 CAN 프로토콜 처리는 CAN Controller가 한다.


### 7. Timer
MCU는 주기적으로 동작해야 한다.
    10 ms 마다 센서 읽기
그래서 Timer가 존재한다.

### 8. ADC (Analog Digital Converter)
센서는 대부분 아날로그 값이다.
    - 온도
    - 전압
    - 압력
MCU는 이 전압을 숫자로 변환해야 한다.

### 9. PWM
모터 속도, LED 밝기 등을 제어할 때 사용한다.

## MCU 동작 과정
자동차 ECU가 부팅되면
``` text
전원 ON

↓

Flash에 있는 프로그램 실행

↓

CPU가 명령 읽음

↓

RAM에 변수 생성

↓

센서 입력 읽음

↓

CAN 송신

↓

모터 제어

↓

반복
```

## CPU와 MCU 차이
MCU는 모든 것이 MCU 내에 있음
| CPU              | MCU                  |
| ---------------- | -------------------- |
| 연산 장치            | 작은 컴퓨터               |
| CPU만 있음          | CPU + Flash + RAM 포함 |
| 외부 RAM 필요        | 내부 RAM 있음            |
| 외부 Flash 필요      | 내부 Flash 있음          |
| Windows/Linux 실행 | 실시간 제어 프로그램 실행       |
| PC, 서버           | 자동차, 가전, IoT         |

## 차량 ECU에서 MCU를 사용하는 이유
자동차에는 여러개의 ECU가 존재한다.
각 ECU마다 특정 기능만 수행하면 되므로, 
저전력, 저비용, 실시간 처리가 가능한 MCU를 사용한다.

## MCU 제조사
자동차에서 많이 사용하는 MCU 제조사는 다음과 같다.

| 제조사                | 대표 제품         |
| ------------------ | ------------- |
| NXP                | S32K, MPC57xx |
| Infineon           | AURIX         |
| Renesas            | RH850         |
| STMicroelectronics | STM32         |
| Texas Instruments  | TMS570        |

# 이해도 체크
Q1. 왜 Python 변수는 RAM에 저장될까?
- RAM은 실행 중인 프로그램과 데이터를 저장하는 메모리
- Python 변수는 계속 값이 변경될 수 있으므로 읽기/쓰기가 매우 빠른 RAM에 저장됨

Q2. CPU가 하는 일은 무엇인가?
- 프로그램 코드를 이해하고, 연산을 수행
- 다른 장치를 제어한다.
  - RAM 에 저장/읽어오라고 명령
  - CAN Controller에게 송신하라고 명령

Q3. CPU와 MCU의 가장 큰 차이점은 무엇인가?
- CPU는 연산/제어를 수행함. Flash, RAM 같은 메모리는 별도로 존재함
- MCU는 이와 달리 MCU 내 Flash, RAM 등이 모두 존재함. 특정 기능을 수행하는 작은 컴퓨터

Q4. 왜 자동차 ECU에는 일반 CPU보다 MCU를 사용할까?
- 자동차 ECU는 특정 기능만 수행하면 되므로, MCU와 같은 경량화된 칩을 사용하면 됨. 
- MCU는 저전력/저비용/작은크기/실시간처리 가능

Q5. Python에서 speed = 100 을 실행하면 Python Interpreter는 무엇을 하는가? CPU는 무엇을 하는가 RAM은 무엇을 하는가? 각자의 역할을 설명할 수 있는가?
- Python Interpreter는 CPU가 코드를 이해할 수 있도록 코드를 변환함
- CPU는 RAM에 있는 코드를 읽고 이해하여 RAM에 speed = 100 저장하도록 명령함
- RAM은 해당 코드가 실행되면 실행코드를 저장해두고, CPU의 명령을 받아 speed라는 변수에 100을 저장함

Q6. MCU 내부에는 CPU 외에 어떤 메모리들이 포함되어 있는가? 각각의 역할은 무엇인가?
- EEPROM : 용량이 작고, 중요한 변수 저장. 비휘발성
- Flash : 프로그램 저장. 비휘발성
- RAM : 실행 중인 변수 저장. 휘발성

Q7. MCU 내부에 CAN Controller가 포함되는 이유는 무엇인가?
- ECU 끼리 CAN 통신을 함. 이를 읽고, 처리해주기 위해 MCU 내부에 포함됨
- CAN Controller는 CAN 프로토콜(Frame 생성, CRC 계산, Arbitration 등)을 처리하기 위한 전용 하드웨어
